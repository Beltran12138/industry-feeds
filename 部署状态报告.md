### 🔴 高优先级 - 必须立即执行

#### 1. 启用 GitHub Actions

**原因：** Vercel Hobby 计划不支持 Cron Jobs，定时任务必须通过 GitHub Actions 运行。

**步骤：**
1. 访问：https://github.com/Beltran12138/industry-feeds/actions
2. 点击绿色按钮 **"I understand my workflows, go ahead and enable them"**
3. 确认所有 workflows 显示为绿色 ✓

**GitHub Actions 将自动执行：**
| 任务 | 频率 | 说明 |
|------|------|------|
| Scrape High Frequency | 每 5 分钟 | Twitter、交易所公告 |
| Scrape Low Frequency | 每 30 分钟 | 新闻网站、博客 |
| Health Check | 每小时 | 数据源健康监控 |
| Daily Report | 每天 18:00 | 企业微信日报 |
| Weekly Report | 每周五 18:00 | 周报总结 |
| Notion Sync | 每 6 小时 | 高价值情报同步 |
| GitHub Export | 每天 00:30 | 日报发布到 Releases |

#### 2. 配置 GitHub Secrets

**原因：** GitHub Actions 需要这些 secrets 才能正常工作。

**步骤：**
1. 访问：https://github.com/Beltran12138/industry-feeds/settings/secrets/actions
2. 点击 **"New repository secret"**
3. 逐个添加以下 secrets：

| Secret Name | Value (示例) | 用途 |
|-------------|--------------|------|
| `DEEPSEEK_API_KEY` | `sk-xxxxxxxxxx` | AI 分析 |
| `WECOM_WEBHOOK_URL` | `https://qyapi.weixin.qq.com/...` | 企业微信推送 |
| `NOTION_API_KEY` | `<your-notion-api-key>` | Notion 同步 |
| `NOTION_DATABASE_ID` | `<your-notion-database-id>` | Notion 数据库 ID |
| `GITHUB_TOKEN` | `<your-github-personal-access-token>` | GitHub API |

## 🔍 故障排查指南

### 问题 1: API 返回 "FUNCTION_INVOCATION_FAILED"

**可能原因：**
- 环境变量未正确加载
- 模块依赖缺失
- Serverless 超时

**解决方法：**
1. 检查 Vercel Logs: https://vercel.com/beltran12138s-projects/alpha-radar/logs
2. 重新部署：`vercel --prod`
3. 确认所有依赖已安装：`npm install`

### 问题 2: GitHub Actions 不运行

**可能原因：**
- Actions 未启用
- Secrets 配置错误
- Workflow 语法错误

**解决方法：**
1. 确认已启用 Actions
2. 检查 secrets 名称是否完全匹配
3. 查看 Actions 日志

### 问题 3: Notion 同步失败

**可能原因：**
- Database ID 错误
- Integration 未分享到数据库
- API 权限不足

**解决方法：**
1. 确认 Database ID 是 32 位十六进制（去掉 `-`）
2. 在 Notion 数据库中分享集成
3. 检查 NOTION_API_KEY 是否正确
