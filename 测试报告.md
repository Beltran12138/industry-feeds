### 1. 企业微信推送去重机制 - **通过**

#### 去重机制（6 层防护）

| 层级 | 机制 | 说明 |
|------|------|------|
| 1️⃣ | **URL 唯一索引** | 数据库层面 URL 去重（UNIQUE 约束） |
| 2️⃣ | **normalized_title 归一化** | 标题标准化后去重（移除大小写、标点、后缀） |
| 3️⃣ | **sent_to_wecom 标记** | 已推送消息标记为 1，后续跳过 |
| 4️⃣ | **源级别冷却时间** | 每个来源独立配置 `pushCooldownHours`（6-48 小时不等） |
| 5️⃣ | **运行时锁** | `pushLock` + `sentThisRun` 防止单次运行内重复 |
| 6️⃣ | **二次 DB 校验** | 推送前再次查询 DB，防止并发场景下的重复 |

#### 关键代码位置

```javascript
// scrapers/index.js:197-252
// 推送前检查
if (item.is_important === 1 && !isAlreadySent) {
  // 检查源级别冷却时间
  if (!canPushMessage(item.source, item.title, item.timestamp, 
                      sourceConfig.pushCooldownHours)) {
    console.log(`[SKIP COOLDOWN] ${item.source}: 冷却期内`);
    continue;
  }
  
  // 运行时锁检查
  if (sentThisRun.has(nTitle) || pushLock.has(nTitle)) {
    continue;
  }
  pushLock.add(nTitle);
  
  // 二次 DB 校验（防并发）
  let dbCheck = db.prepare(
    'SELECT sent_to_wecom FROM news WHERE (url = ? OR normalized_title = ?) 
     AND sent_to_wecom = 1'
  ).get(item.url, nTitle);
  
  if (dbCheck) {
    continue; // 已在 DB 中标记为已发送
  }
  
  // 先保存再发送（原子操作）
  await saveNews([item]);
  await sendToWeCom(item);
  updateSourcePush(item.source, item.timestamp, item.title);
}
```

```javascript
// report.js → sendReportToWeCom()
async function sendReportToWeCom(reportContent, reportType) {
  // 自动分段（每段 < 4096 字符）
  const segments = splitReportContent(formattedContent, 4096 -200);
  
  for (let i = 0; i < segments.length; i++) {
    await axios.post(WECOM_WEBHOOK_URL, {
      msgtype: 'markdown',
      markdown: { content: segments[i] }
    });
    
    // 多段之间间隔 1 秒
    if (i < segments.length -1) {
      await new Promise(r => setTimeout(r, 1000));
    }
  }
}
```

| 任务 | Cron 表达式 | UTC 时间 | 北京时间 | 执行日期 |
|------|------------|---------|---------|---------|
| 爬虫 | `*/15 * * * *` | 每 15 分钟 | 每 15 分钟 | 每天 |
| 日报 | `0 10 * * 1-4` | 10:00 | 18:00 | 周一至周四 |
| 周报 | `0 10 * * 5` | 10:00 | 18:00 | 周五 |
